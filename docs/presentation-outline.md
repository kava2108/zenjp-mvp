### スライド 3: ZenJPの解決策

**レイアウト:** 3つのカラム（3軸の説明）

#### ビジュアル指示
- **3つのカラム:** Value / Growth / Momentum
- **各カラム:** アイコン＋説明テキスト
- **下部:** 統合スコア＋ランク表示

#### コンテンツ

```
┌──────────────┬──────────────┬──────────────┐
│   VALUE      │   GROWTH     │  MOMENTUM    │
│   割安性     │   成長性     │   勢い       │
├──────────────┼──────────────┼──────────────┤
│ PER/PBR      │ 売上成長率   │ RSI          │
│ 配当利回り   │ 利益成長率   │ 株価変動     │
│              │ ROE/営業利益 │ 取引量       │
│              │              │              │
│ 📊 62点      │ 📈 88点      │ 📉 50点      │
└──────────────┴──────────────┴──────────────┘

業種別重み付け適用

┌──────────────────────────────────┐
│  総合スコア: 68.67点  ランク: B  │
│  市場平均(50点) より高い!        │
└──────────────────────────────────┘
```

#### Speaker Notes
> ZenJPの解決策はシンプルです：**複数の指標を1つのスコアに統合する**。
> 
> 具体的には、3つの投資軸で企業を評価します。
> 
> まず、**Value（割安性）**。
> PER、PBR、配当利回りを組み合わせて、
> 「この企業は安く買えるか」を0-100点で評価します。
> 
> 次に、**Growth（成長性）**。
> 売上成長率、利益成長率、ROEから、
> 「この企業は今後成長するか」を評価します。
> 
> 最後に、**Momentum（勢い）**。
> RSI、株価変動率、取引量から、
> 「この株価に勢いがあるか」を評価します。
> 
> これら3つの軸を、業種別の重み付けを加えて統合し、
> **総合スコア（0-100点）**と**ランク（A～D）**を算出します。
> 
> さらに、市場平均（50点）との比較で、
> 「この銘柄は市場より魅力的か」が一目瞭然です。

### スライド 5: アーキテクチャ

**レイアウト:** システム構成図（3層構成）

#### ビジュアル指示
- **フロントエンド層:** Next.js 14 / TypeScript / Tailwind CSS
- **バックエンド層:** FastAPI / Python / SQLAlchemy
- **データ層:** PostgreSQL / Redis（予定）
- **矢印:** 各層間の通信を表示

#### コンテンツ

```
┌────────────────────────────────────────────┐
│         フロントエンド                     │
│  Next.js 14 / TypeScript / Tailwind CSS   │
│  React Server Components, SSR対応         │
│                  │                        │
│                  ▼                        │
├────────────────────────────────────────────┤
│          バックエンド（API）               │
│   FastAPI / Python / SQLAlchemy           │
│   - RESTful API設計                       │
│   - 非同期処理対応                        │
│   - OpenAPI 自動ドキュメント              │
│                  │                        │
│                  ▼                        │
├────────────────────────────────────────────┤
│            データベース層                  │
│   PostgreSQL（主DB）                      │
│   Redis（キャッシング、予定）             │
│                                           │
│   構成：Docker Compose                   │
│   環境：本番対応（EC2/RDS可能）           │
└────────────────────────────────────────────┘
```

#### Speaker Notes
> 次に、ZenJPの技術アーキテクチャをご説明します。
> 
> システムは3層構成です。
> 
> **フロントエンド層**は、Next.js 14を採用しました。
> モダンなReact開発が可能で、
> サーバー側レンダリング（SSR）対応で、
> SEO最適化やパフォーマンスが優れています。
> 
> **バックエンド層**は、FastAPI（Python）です。
> 非同期処理対応で、
> 同時に多数のリクエスト処理が可能。
> OpenAPI自動ドキュメント機能で、
> API仕様書の自動生成も実現しています。
> 
> **データベース層**は、PostgreSQLを採用。
> 金融データの信頼性が必須なため、
> 商用グレードのRDBMSが必要です。
> 
> インフラは、Docker Composeで統合管理。
> 開発環境と本番環境の差異を最小化しています。
